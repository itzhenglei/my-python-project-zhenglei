# AGENTS.md — 协作与编码习惯知识库

本文档由助手维护，用于**长期记录和更新**编码习惯、测试风格与协作偏好。助手应在后续对话中**主动从用户表述和新代码中提取**新习惯，并在适当时**补充或调整**本文件。

配套：**`.cursor/rules/`** 中的规则用于在 Cursor 内始终拉取关键约束。

---

## 1. 语言与沟通

- 助手使用**简体中文**回复。
- 复杂逻辑优先在**对话中**讲清楚。
- 涉及行为变化时，先**方案对比**再实现。

---

## 2. 改代码与文档（与 Cursor 规则对齐）

- **新建文件**：可直接创建，无需事先征得同意。
- **修改或删除**已有文件：先说明将改哪些文件、改什么，用户用「可以改」「授权」等确认后再执行。
- **说明类 Markdown**（如项目总览文档）：用户**主动要求**时才大段新增或更新；**本 AGENTS.md** 除外（由助手按约定维护）。
- **快乐 8 项目**：源码注释「能让人看懂即可」；邮件 **`generate_email_content`** 等大段 HTML/CSS **f-string 内不要 `#` 注释**。
- **号序（方案 A）**：主流程先取胆再 `sorted`，邮件/JSON/打印一致（详见项目 `文档-程序逻辑与协作习惯.md`）。

---

## 3. 导入习惯

- **工程内模块**（如 `utils.common_page`、`page.xxx`）：为避免循环导入，可在**函数**或 **`__init__`** 内按需导入。
- **非工程内模块**（标准库、第三方库）：在**文件顶部**导入。

---

## 4. 文件与配置习惯

- **`pytest.ini`**：`addopts`、`markers` 集中配置，并与 **allure** 报告输出目录配合。
- **`config.json`**：远端服务器、ipa 路径等环境配置集中管理。
- **文件名**：有意义的**英文**（如 `test_login.py`、`base_page.py`、`log_keyword.py`），**不使用中文文件名**。

---

## 5. Git 操作习惯

- **修改前必须征得同意**：任何改动（代码、配置等）先说明方案，确认后再执行。
- **恢复/撤销也需同意**：`git reset`、`git revert`、回退等先说明并征得同意。
- **提交流程**：
  - 说明要执行的命令及作用；
  - 说明本次提交涉及的文件；
  - 明确区分 **AI 完成** 与 **人为编写**；
  - commit message 中写明 AI 完成的部分；
  - 用户确认后再执行 `git commit`；
  - `git push` 前单独征求同意。
- **提交范围**：规范中文 commit；仅提交约定文件；不把本地预测生成的 html/json 等随意入库，除非用户要求。

---

## 6. 对助手的行为要求

- **主动记录习惯**：从用户表述中主动提取操作习惯与协作偏好，写入**本文件**，不要等用户反复提醒。
- **更新习惯后必须告知**：每次在本文件中新增或修改习惯条目后，在**当次回复中明确告知**用户（例如「已记录到 AGENTS.md」）。
- **主动完成可自动化事项**：在能力范围内不必等用户逐条下指令。
- **方法治理原则**：无效且未被引用的方法及时删除；仅单一场景且实现错误时，确认不影响其他代码后可就地修正，不保留无效试错方法。
- **重构/批量改动**：先生成临时完整文件（如 `xxx_TEMP.py`），用户确认后再覆盖原文件；完成后**删除**临时文件。
- 可在合适时新建辅助类或说明文件以增强协作。
