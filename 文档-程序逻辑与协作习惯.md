## 1. 项目概述

- **主程序**：`kl8ycshunew.py` —— 快乐 8「智能预测」：抓取公开网页/API → 多维度评分选号 → 生成 HTML 邮件 → 发信并保存 HTML/JSON。
- **依赖**：见 `requirements.txt`（`requests`、`beautifulsoup4`、`zmail`、`schedule`）。
- **其他**：`webui.py` 当前基本为空；运行时产物可能包含 `kl8_intelligent_prediction*.html`、`kl8_intelligent_prediction.json`（通常不入库）。

## 2. 端到端程序逻辑（主流程）

入口：`python kl8ycshunew.py` → `main()` → **`run_scheduler()`**（常驻：注册每天 **17:30** 执行 `process_and_send_email`，并**立即先跑一轮**，随后 `while True` 每约 60 秒 `schedule.run_pending()`）。

核心业务函数：**`process_and_send_email()`**，顺序大致为：

| 步骤 | 作用 |
|------|------|
| 1 | **`DataFetcher`**：从 `KL8_HTML_URL` 拉推荐页 HTML，解析多期「期号、日期、已开奖 20 码、五类推荐」；从 `KL8_API_URL` 拉开奖 JSON，**`parse_api_data`** 得到 `historical_draws`、`sorted_issues`。 |
| 2 | **`PeriodicityAnalyzer`**：在多种「连续期数」下滑动窗口，看相邻期号码集合重复程度，选 **stability** 最好的窗口长度 → **`optimal_periods`**（回测用多长）。 |
| 3 | **`BacktestOptimizer`**：多组维度权重配置回测，选命中率更好的一组 → **`optimal_weights`**，并产出回测统计（含高/低频推荐命中率等）。 |
| 4 | **`IntelligentPredictor`**：用最优权重，对 **`all_periods_data` 前 15 期** 各做 **`predict_for_period(..., count=10)`**；预测只从**当期五类推荐合并池**里按**综合得分**取前 10。**对内**得分顺序决定胆码候选。 |
| 5 | 对**列表第一条（最新一期）**再预测一次；**先**用分数序列表取胆码 **`[:2]`**，**再**对 10 个号码 **`sorted()`** 得到对外展示顺序；胆码集合不变，**`dan_codes` 再 `sorted()`** 仅影响显示顺序。 |
| 6 | **`generate_email_content`**：拼完整 HTML（含大段内嵌 CSS 的 f-string）；**勿在 f-string 内写 `#`，否则会进邮件或破坏语法**。 |
| 7 | 写本地带时间戳的 HTML、**`send_email`**；成功则写 **`kl8_intelligent_prediction.json`**。 |

**预测语义要点**：

- **`predict_for_period`** 返回的列表顺序为**分数从高到低**，不是号码升序。  
- **对外统一号序（方案 A）**：在 **`process_and_send_email`** 里，**先定胆再 `sorted(predicted_numbers)`**；**`all_predictions` 各期**存 **`sorted(predicted)`**；最新期用本轮结果 **回写 `all_predictions[当期]`**，避免与步骤 4 重复算同一期的列表不一致。  
- **回测/命中**：多用 **集合交集**，与列表顺序无关。

## 3. 主要类与辅助函数（速查）

- **`DataFetcher`**：`fetch_html_recommend_data`、`fetch_lottery_api_data`、`parse_api_data`
